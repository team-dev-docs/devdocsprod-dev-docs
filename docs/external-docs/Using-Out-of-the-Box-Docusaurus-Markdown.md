# Using Out of the Box Docusaurus Markdown

## Using Docusaurus Markdown

If for some docs or pages you prefer to use the markdown/mdx functionality out the box with Docusaurus you can just add plain markdown files. Dev-Docs is built around Docusaurus, so you are not really losing any of the main Docusaurus functionality!

## Dev-Docs Markdown vs. Standard Docusaurus Markdown

While Dev-Docs provides enhanced features for documentation, there may be times when you want to use standard Docusaurus markdown. Here's a quick comparison:

1. **Dev-Docs Markdown:**
   - Offers additional components for interactive documentation
   - Provides a more structured approach to content creation
   - Ideal for complex documentation with interactive elements

2. **Standard Docusaurus Markdown:**
   - Simpler and more lightweight
   - Familiar to those already comfortable with basic markdown
   - Suitable for straightforward content without need for advanced features

Choose Dev-Docs markdown when you need enhanced interactivity and structure in your documentation. Opt for standard Docusaurus markdown for simpler pages or when you want to quickly add content without additional components.

<br></br>

Checkout the core [Docusaurus Markdown Syntax](https://docusaurus.io/docs/markdown-features) for more info on standard Docusaurus markdown features.

Remember, you can mix and match both approaches in your project to best suit your documentation needs!
