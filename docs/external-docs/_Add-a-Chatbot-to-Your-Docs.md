# Add a Chatbot to Your Docs

### Step 1: Find Settings Icon

Sign in to [dev-docs.io](https://dev-docs.io), go to External Docs, and click on the gear icon on the right side of the screen. 

![add_a_chatbot_to_your_docs/step_5](/img/add_a_chatbot_to_your_docs/step_5.png)

### Step 2: Create Chat Bot

Select GPT Github Setup and click Create Chat button. 

![add_a_chatbot_to_your_docs/step_7](/img/add_a_chatbot_to_your_docs/step_7.png)

## Step 3: Add to Your Docs

Click the "Add to Your Docs" button.

![add_a_chatbot_to_your_docs/step_10](/img/add_a_chatbot_to_your_docs/step_10.png)

### Step 4: Customize Your Chatbot

After adding the chatbot, you can customize it for your specific codebase:

1. Go to the Chatbot Settings page
2. Upload relevant documentation or code snippets
3. Configure the chatbot's knowledge base
4. Set up custom responses for frequently asked questions

### Step 5: Use Cases and Best Practices

Implement your chatbot effectively with these use cases:

- Answer common developer questions
- Provide quick access to API documentation
- Assist with troubleshooting and debugging
- Guide users through complex workflows

For best results, regularly update your chatbot's knowledge base as your codebase evolves.

### Step 6: Understanding Limitations

While powerful, chatbots have some limitations to keep in mind:

- May not always understand complex context
- Requires regular updates to stay current
- Cannot replace human support entirely

By understanding these limitations, you can set appropriate expectations and use the chatbot as a complementary tool to your existing documentation and support channels.
