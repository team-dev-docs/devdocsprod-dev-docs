# How to Get Started With User-Facing Docs

## Intro to Dev-Docs User-Facing Documentation

At Dev-Docs we believe in documentation as code. Using Dev-Docs you can start generating user-facing documentation directly from your existing codebase. Configure what exactly you want documented and start generating first drafts of new user-facing docs in just minutes. Your docs are generated as Markdown files and live with your codebase. You always have the final say and can edit or use the Markdown files as you see fit. 

## Create Your Dev-Docs User-Facing Documentation Repo 🦾

If you are new to Dev-Docs, just go to [dev-docs.io](https://dev-docs.io) and sign up with GitHub. Click the User-facing docs tab and Dev-Docs will create a tailored Dev-Docs GitHub repo and Dev-Docs GitHub codespace. From there you can start editing your docs! Alternatively, you can bring your existing docs repo to using Dev-Docs where your existing docs already live. 

## Setting Up Dev-Docs for Your Project

1. **Connect Your Repository**: Once you've created your Dev-Docs account, connect it to your GitHub repository containing the code you want to document.

2. **Configure Documentation Settings**: Use the Dev-Docs dashboard to specify which parts of your codebase you want to document. You can select specific folders, file types, or even individual files.

3. **Choose Documentation Style**: Select from various documentation styles that best fit your project's needs. Dev-Docs supports multiple formats including API references, user guides, and tutorials.

4. **Generate Initial Docs**: Click the 'Generate Docs' button to create the first draft of your user-facing documentation based on your code and configuration.

5. **Review and Edit**: Once generated, review the Markdown files in your connected GitHub repository. You can make edits directly in GitHub or clone the repository to your local machine for more extensive changes.

6. **Continuous Updates**: As you update your code, Dev-Docs can automatically regenerate documentation, ensuring your user-facing docs stay in sync with your latest code changes.

Remember, Dev-Docs is here to accelerate your documentation process, not replace human insight. Always review and refine the generated content to ensure it meets your users' needs.
