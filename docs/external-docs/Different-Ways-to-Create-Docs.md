## Using the VS Code Extension Editor

This is a full rich text editor built into VS Code using the Dev-Docs VS Code extension. It is very similar to what you could expect from a notion type interface. It can be accessed at any time by right clicking any code file and selecting "Open Devdocs Markdown." This editor has built in benefits including:

- AI helpers for content generation and improvement
- Out-of-the-box styling options
- Pre-configured component selections

These features eliminate the need to remember shorthand or complex markdown syntax, making it ideal for users who prefer a visual editing experience.

## Using Dev-Docs Markdown

Dev-Docs has its own custom markdown that gives you more flexibility than the out-of-box markdown in Docusaurus. It allows you to style your markdown and use custom components without needing to use React. Some advantages include:

- Extended syntax for advanced formatting
- Easy integration of custom components
- Simplified table and diagram creation

This approach is perfect for users who want more control over their document structure while still maintaining the simplicity of markdown.

## Using Docusaurus Markdown

At its core, Dev-Docs user-facing documentation framework is built around all the great functionality of Docusaurus. If you want to use out-the-box Docusaurus features and markdown syntax, you can do that. Benefits of this method include:

- Familiarity for users already experienced with Docusaurus
- Access to all standard Docusaurus plugins and themes
- Seamless integration with existing Docusaurus projects

This method is best for those who want to stick with traditional markdown and leverage the full power of Docusaurus.

## Comparison and Use Cases

| Method | Best For | Learning Curve | Customization |
|--------|----------|----------------|---------------|
| VS Code Extension | Visual editors, beginners | Low | Medium |
| Dev-Docs Markdown | Advanced users, custom needs | Medium | High |
| Docusaurus Markdown | Docusaurus experts, standard docs | Low | Medium |

Choose the method that best aligns with your team's skills, project requirements, and desired level of customization. For complex projects, you may even use a combination of these approaches to maximize efficiency and flexibility.
